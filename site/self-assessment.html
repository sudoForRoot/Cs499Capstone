CS 499 · Final Project

Professional Self-Assessment

Completing the Bachelor of Science in Computer Science at Southern New Hampshire University has been a process of building capability layer by layer — from foundational programming concepts through object-oriented design, data structures, software testing, and finally into this capstone experience that required synthesizing all of it into a coherent, professional body of work.

The three enhancements completed for this capstone were all applied to the same artifact: the Grazioso Salvare Rescue Animal Dashboard, originally developed in CS 340. Applying all three enhancement categories to a single project was a deliberate choice that demonstrates not only competency in each area individually, but the ability to reason about how those areas interact and make targeted improvements that are good for the whole system.

Collaborating in a Team Environment

Software development is fundamentally a collaborative discipline. In CS 250: Software Development Lifecycle, I worked within a simulated Agile team environment, participating in sprint planning, daily standups, and retrospectives. In CS 320: Software Testing, Automation, and Quality Assurance, I developed an appreciation for how testing functions as a form of team communication — a unit test suite is a specification that any team member can read to understand what a function is supposed to do.

This perspective directly influenced the capstone: every enhancement module includes a comprehensive test suite (33 tests for the algorithms module, 21 for the database module) not as a formality, but as documentation of intent that a future collaborator can depend on.

Communicating with Stakeholders

The Grazioso Salvare Dashboard was built for an animal rescue organization whose staff are domain experts in rescue operations, not software engineers. Every design decision had to be justified in terms of what a field coordinator would need. The narratives written for each milestone extend this practice — they are written to be accessible to a technical hiring manager who may not be a MongoDB specialist.

Data Structures and Algorithms

CS 260 established the theoretical foundation for the algorithms enhancement in Milestone Three. What the capstone added was the experience of selecting and implementing those structures for a specific, real problem. The decision to use a hash-map index for O(1) lookups, an inverted index for O(k) breed filtering, a min-heap for ranked candidate retrieval, and binary search for age-range filtering required analyzing the dashboard's access patterns and matching those patterns to the right structures — bridging the gap between knowing a data structure and knowing when to use it.

Software Engineering and Database

The software design enhancement applied principles from IT 315 and CS 250 to restructure the original CS 340 codebase: separating database access, application logic, security, and utilities into distinct, independently testable modules reflecting the single-responsibility principle. Environment-variable-driven configuration means the same code runs in development, staging, and production without modification.

The database enhancement introduced a compound index that converts full collection scans to index scans, JSON Schema validation enforced at write time, an explicit connection pool with health-check logic, server-side aggregation pipelines that reduce network transfer, and query performance logging that surfaces COLLSCAN operations during development.

Security

A security mindset was woven through all three enhancement categories. In the software design enhancement, security.py introduced input sanitization, MongoDB query validation rejecting dangerous operators ($where, $eval), password hashing with SHA-256 and salt, and HTTP security headers. In the database enhancement, JSON Schema validation prevents malformed documents at write time, and connection pooling limits concurrent connection exposure. These controls implement defense in depth — multiple independent layers that collectively protect data integrity.

Artifact Summary

The three enhancements form a complete, layered improvement to a single coherent project. The software design foundation established clean architecture; the algorithms enhancement built an efficient in-memory query layer on top of it; the database enhancement completed the persistence strategy with server-side computation and enforced data integrity. Together they demonstrate that I can reason about an entire software system and make targeted, justified improvements to each layer without breaking the others.
